> Generated by `examples/run_h2.py`. The moat (a public verified-skill registry) is real only if verified skills RE-VERIFY across tenants. This measures it on a live-built corpus.

## Interpretation

Skills that re-verify everywhere are fungible — a genuine cross-tenant network effect. Skills that re-verify only where a tenant's rules happen to match are lock-in, not a moat. §8.7 says measure THIS before building the public registry.
